# ☯ 本源实践规划 — 真实复杂 PCB 项目全链路复刻推演

> 正本清源: 本体系不是"用固定模板去测试 KiCad 能力"的系统 — 那是 KiCad 自己的活。
> 本体系的本源是: **Agent 作为核心, 参照网络上真实、复杂、优质的开源 PCB 项目
> (电脑主板、手机/掌机主板级), 从零开始全链路复刻推演 (原理图 → ERC → 网表 →
> 建板 → 布线 → DRC → 制造输出), 在实践过程中暴露工具体系的缺陷, 并持续完善
> 工具本身** — 创造新工具、融合旧工具、提升效率与能力。道法自然, 无为而无不为。

## 错误认知的剔除

| 旧 (错误) | 新 (本源) |
|---|---|
| 固定死板的模板矩阵当"测试" | 真实开源复杂项目的全链路复刻实践 |
| 验证 KiCad 本身能不能跑 | 验证 **Agent + 工具体系** 能否全链路产出成品 |
| 一次性 scoreboard 数字 | 每轮实践 = 暴露缺陷 → 修复/演化工具 → PR → 下一轮 |

## 实践环 (每一轮大规模实践的恒定形态)

```
选参照 (网络优质开源复杂 PCB 项目, 电脑主板 / 手机主板级)
  → 从零全链路复刻: ERC → 网表 → 建板(封装自愈) → freerouting → DRC → Gerber/BOM/CPL
  → 记录每一阶段的原始结果 (chain_result.json)
  → 归因每个失败/脏点: 是工具缺陷? 工具搭配逻辑缺陷? 还是项目本身边界?
  → 工具缺陷 → 当场修复 → 回归 → PR (自动合并)
  → 沉淀到文档 (缺陷→修复→证据), 进入下一个参照项目
```

## 工具演化的判据

- 每个修复必须来自实践中真实暴露的缺陷 (不造伪需求)。
- 修复必须存在性门控 (existence-gated), 永不瞎猜 (如封装自愈只采纳库内真实存在者)。
- 闭环收尾必须单调不劣化 (finisher 仅在 DRC 严格改善时采纳候选板)。
- 每轮实践的证据 (板名、DRC 数字、缺陷→修复对照) 都写入文档, 可复现。
